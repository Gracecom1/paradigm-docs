## 前提条件

**あなた自身の PC にインストールするものは何もありません。**  
エージェントは Hetzner サーバー上で Docker コンテナとして動作します。

| 必要なもの | 説明 |
|-----------|------|
| Twenty CRM アカウント | `twenty.paradigmjp.com` にログインできること |
| Hermes チャット | エージェントへの実行指示を出すインターフェース |
| インターネット接続 | 上記 2 つにアクセスできれば OK |

---

## 操作の流れ

```
Twenty CRM（twenty.paradigmjp.com）
    │  企業を登録 → ステータスを「送信待ち / 未対応」に
    │
Hermes チャット
    │  「エージェント --max 5 で実行して」
    │
    ▼
Hetzner サーバー
    │  Agent Runner Webhook (systemd :4444) が受信
    │  → docker run outbound-form-agent:v2 --max 5
    │  → Twenty CRM から企業取得 → フォーム送信 → 結果書き戻し
    │  → コンテナ自動終了
    │
Twenty CRM
      結果を確認（ステータスが自動更新されている）
```

---

## Hermes への指示例

```
# 安全確認（送信なしでテスト）
「エージェントを --dry-run で実行して」

# 1 社テスト
「エージェントを --single で実行して」

# 5 社だけ本番実行
「エージェントを --max 5 で実行して」

# 特定企業だけ
「エージェントを --company "株式会社ABC" で実行して」
```

## サーバー構成（参考）

すべて Hetzner 側にセットアップ済み。あなたが触る必要はありません。

| コンポーネント | 場所 | 状態 |
|---------------|------|------|
| Docker イメージ | `outbound-form-agent:v2` | ✅ ビルド済み |
| Agent Runner | systemd `agent-runner.service` | ✅ 常駐 |
| .env | `/opt/outbound-form-agent-v2/.env` | ✅ 設定済み |
| DeepSeek API | API キー設定済み | ✅ 有効 |

## 結果の確認方法

1. `twenty.paradigmjp.com` を開く
2. 企業一覧で `paradigmSalesStatus` 列を確認
3. ステータスが変化している：

| ステータス | 意味 | 次のアクション |
|-----------|------|---------------|
| `送信済み / 未対応` | フォーム送信成功 | 企業からの返信を待つ |
| `手動確認 / 未対応` | 自動送信失敗 | `paradigmLastError` 列の理由を確認 |
| `ブロック / 未対応` | CAPTCHA 等でブロック | メール等の別手段で連絡 |

## トラブルシューティング

### エージェントが動かない

Hermes に以下を伝える：
```
「agent-runner の状態を確認して」
```

Hermes が SSH でサーバーをチェックし、問題があれば自動修復する。

### 全社スキップされる

Twenty CRM に「送信待ち / 未対応」かつ `formUrl` が入力された企業が存在するか確認。

### DeepSeek API エラー

残高・API キーの問題。Hermes に伝えれば確認・対応可能。
