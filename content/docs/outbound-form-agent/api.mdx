操作はすべて Hermes チャット経由。コマンドラインや Git 操作は一切不要です。

## Hermes チャット指示一覧

### マルチソース収集 (v3 新機能)

| 指示 | 説明 |
|------|------|
| `--collect-sources us --max 20 で実行して` | 米国ドメインを 20 件収集 → 審査 → CRM |
| `--collect-sources jp --max 50 で実行して` | 日本ドメインを 50 件収集 |
| `--zone com --max-scan 100 で実行して` | .com ゾーンファイルから収集 |

### フォーム送信

| 指示 | 説明 |
|------|------|
| `--max 5 で実行して` | 最大 5 社を本番送信 |
| `--max 5 --force で実行して` | 手動確認企業も強制処理 |
| `--dry-run で実行して` | 送信せず対象企業リストのみ表示 |
| `--single-url "https://..." --company "社名" でテストして` | 1URL を直接テスト |

### 履歴確認

| 指示 | 説明 |
|------|------|
| `--history を見せて` | 直近 10 件の送信履歴 |
| `--history 30 を見せて` | 直近 30 件 |
| `履歴サマリを見せて` | 成功率・CAPTCHA 率サマリ |

### 管理

| 指示 | 説明 |
|------|------|
| `エージェントの状態を確認して` | Docker コンテナ・イメージ状態 |
| `エージェントを再ビルドして` | git pull → docker build → デプロイ |

## 全 CLI オプション一覧

| オプション | 値 | デフォルト | 説明 |
|-----------|-----|----------|------|
| `--collect-sources <region>` | us/jp/au/gb/... | — | マルチソース収集 (v3) |
| `--zone <tld>` | com/net/au/com/uk | — | ゾーンファイルハント |
| `--max-scan <n>` | 数字 | 500 | ゾーンハントの最大スキャン数 |
| `--max <n>` | 数字 | 全件 | 最大処理数 |
| `--dry-run` | — | — | 送信せず検出のみ |
| `--force` | — | — | 手動確認企業も強制処理 |
| `--single-url <url>` | URL | — | 1URL 直接テスト |
| `--company <name>` | 文字列 | — | 企業名指定/フィルタ |
| `--timeout <ms>` | ミリ秒 | 120000 | 1社あたりタイムアウト |
| `--concurrency <n>` | 数字 | 2 | 同時実行数 (最大8) |
| `--history [n]` | 数字 | 10 | 送信履歴表示 |
| `--history-summary` | — | — | 履歴サマリ |
| `--collect <query>` | 文字列 | — | Google 検索収集 |
| `--debug` | — | — | デバッグログ有効化 |
| `--help` | — | — | ヘルプ表示 |

## Docker コマンド（参考）

以下は Hermes が内部的に実行するコマンド。

```bash
# マルチソース収集
docker run --rm --env-file /root/outbound-agent-v3/.env \
  -v /var/run/docker.sock:/var/run/docker.sock \
  outbound-form-agent:v3 --collect-sources us --max 20

# ゾーンファイルハント
docker run --rm --env-file /root/outbound-agent-v3/.env \
  -v /var/run/docker.sock:/var/run/docker.sock \
  outbound-form-agent:v3 --zone com --max-scan 100

# フォーム送信
docker run --rm --env-file /root/outbound-agent-v3/.env \
  outbound-form-agent:v3 --max 5 --force

# 単一 URL テスト
docker run --rm --env-file /root/outbound-agent-v3/.env \
  outbound-form-agent:v3 --single-url "https://example.com/contact" \
  --company "Example Corp"

# 履歴確認
docker run --rm outbound-form-agent:v3 --history 10
```

## エラー時の対処

| 現象 | 原因 | Hermes への指示 |
|------|------|----------------|
| 全社スキップ (収集) | 品質ゲートで除外 | `--max` を増やす |
| 全社スキップ (送信) | CRM に送信待ち企業なし | `--dry-run` で確認 |
| DeepSeek エラー | API キー・残高問題 | `DeepSeek API の状態を確認して` |
| コンテナ起動失敗 | イメージ破損 | `エージェントを再ビルドして` |
| MassDNS エラー | Docker ソケット未マウント | `-v /var/run/docker.sock:...` を確認 |
| CRM 400 エラー | 列挙型の値が不正 | Hermes に伝えれば自動修正 |
