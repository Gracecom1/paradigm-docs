## 全体像

Outbound Form Agent v3 は **マルチソース収集** と **フォーム自動送信** の 2 つのパイプラインを持つワンショット Docker コンテナです。

```
Hermes チャット
    │  「--collect-sources us --max 20」
    ▼
Hetzner サーバー (SSH)
    │  docker run --env-file .env -v /var/run/docker.sock:/var/run/docker.sock
    │  outbound-form-agent:v3 --collect-sources us --max 20
    ▼
Docker コンテナ（処理後即終了）
    │
    ├──▶ マルチソース収集パイプライン
    │      ├─ crt.sh Certificate Transparency
    │      ├─ urlscan.io スキャン結果
    │      ├─ Whoxy WHOIS API
    │      ├─ ICANN CZDS ゾーンファイル
    │      └─ Tranco Top 1M リスト
    │
    ├──▶ 重複除去 + DNS 解決
    │      ├─ ドメイン正規化・重複除去
    │      └─ MassDNS (massdns:local) 並列解決
    │
    ├──▶ HTTP スキャン + 技術検出
    │      ├─ 並列 HTTP HEAD/GET (concurrency: 50)
    │      └─ Wappalyzer CMS 検出
    │
    ├──▶ フォーム URL 自動検出
    │      └─ 17 パス HEAD プローブ (/contact, /form, ...)
    │
    ├──▶ DeepSeek 企業審査
    │      ├─ スコアリング (0-100)
    │      └─ 業種判定
    │
    ├──▶ 品質ゲート
    │      ├─ qualScore >= 30 OR CMS 検出 → CRM 登録
    │      └─ それ以外 → スキップ
    │
    ├──▶ CRM 重複チェック
    │      └─ 既存ドメイン照会 → スキップ
    │
    └──▶ Twenty CRM 登録
           └─ POST /rest/companies (全フィールド完備)
```

## マルチソース収集パイプライン詳細

### 1. ソース並列収集

5 ソースを同時実行。各ソースは独立してレート制限制御。

| ソース | 方式 | レート制限 |
|--------|------|-----------|
| crt.sh | Certificate Transparency API | 1 req/2s |
| urlscan.io | スキャン結果 API | 1 req/5s |
| Whoxy | WHOIS API | 1 req/10s |
| CZDS | ICANN ゾーンファイル (.com/.net/.shop/.company) | — |
| Tranco | Top 1M ドメインリスト | — |

### 2. 重複除去・DNS 解決

```
生ドメイン
  → 正規化 (lowercase, www除去, 末尾/除去)
  → ドメイン単位の Set で重複除去
  → MassDNS 並列解決 (concurrency: 50)
  → 解決済みドメインのみ次工程へ
```

MassDNS は `massdns:local` イメージ (alpine:3.19 + ソースコンパイル) を Docker-in-Docker で実行。

### 3. HTTP スキャン + 技術検出

```typescript
// 並列スキャン
const results = await scanDomains(domains, {
  concurrency: 50,
  timeoutMs: 5000,
});
// → { domain, statusCode, server, cms, reachable }

// Wappalyzer CMS 検出
const cms = await detectCMS(domain);
```

### 4. フォーム URL 自動検出

17 の共通コンタクトパスを HEAD リクエストで並列プローブ:
`/contact`, `/contact-us`, `/form`, `/inquiry`, `/support`, ...

### 5. DeepSeek 企業審査

```typescript
const result = await qualifyLeads([
  { websiteUrl, companyName, cms, ... }
]);
// → { qualification: { totalScore, industry, ... } }
```

### 6. 品質ゲート

```
qualScore >= 30 OR CMS 検出 → CRM 登録
それ以外 → スキップ (ゴミデータ防止)
```

### 7. CRM 重複チェック

CRM 登録前に既存ドメインをバッチ照会。既存の場合はスキップ。

### 8. Twenty CRM 登録

```json
{
  "name": "...",
  "domainName": { "primaryLinkUrl": "...", "primaryLinkLabel": "..." },
  "paradigmCountryName": "米国",
  "paradigmRegionName": "US",

  "paradigmDataSources": "crt.sh, urlscan.io",
  "paradigmDataStatus": "multi_source_collected",
  "paradigmNextAction": "tech_check",
  "paradigmFormUrl": { "primaryLinkUrl": "...", ... }
}
```
