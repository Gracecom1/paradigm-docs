## 全体像

Outbound Form Agent v2 は **「Hermes 指示 → Docker 起動 → 処理 → Twenty 書き戻し」** のワンショットモデルです。

```
Hermes チャット
    │  「エージェント --max 5 で実行」
    ▼
Agent Runner Webhook (:4444, systemd 常駐)
    │  docker run --rm --env-file .env outbound-form-agent:v2 --max 5
    ▼
Docker コンテナ（処理後即終了）
    │
    ├──▶ Twenty CRM REST API
    │      └─ GET /rest/companies → 「送信待ち」企業を取得
    │
    ├──▶ Stagehand v3 LOCAL（CDP Chrome）
    │      ├─ page.goto(formUrl)
    │      ├─ CAPTCHA 検出（事前チェック）
    │      ├─ DeepSeek API → 営業文生成
    │      ├─ Stagehand agent.execute(instruction) → フォーム入力・送信
    │      └─ 結果判定
    │
    └──▶ Twenty CRM REST API
           └─ PATCH /rest/companies/{id} → 結果を書き戻し
```

## 1 社あたりの処理詳細

### 1. 企業情報の取得

Twenty CRM から処理対象企業を取得。

```http
GET /rest/companies?limit=10000
```

クライアント側で「送信待ち / 未対応」かつ `formUrl` ありの企業をフィルタ。

### 2. Stagehand v3 LOCAL ブラウザ起動

Stagehand v3 は CDP（Chrome DevTools Protocol）で Chrome を直接制御。Playwright レイヤーは不要。

```typescript
const stagehand = new Stagehand({
  env: "LOCAL",
  localBrowserLaunchOptions: {
    headless: true,
    executablePath: process.env.CHROMIUM_PATH,
    args: ["--disable-blink-features=AutomationControlled", "--no-sandbox"],
  },
  model: {
    modelName: "deepseek/deepseek-chat",
    baseURL: "https://api.deepseek.com/v1",
  },
});
await stagehand.init();
```

### 3. フォームページへのアクセス

Stagehand の内部 Page（CDP-backed）でアクセス。

```typescript
const page = stagehand.context.activePage();
await page.goto(formUrl, { waitUntil: "networkidle", timeoutMs: 20_000 });
```

### 4. CAPTCHA 事前チェック

agent を走らせる前に高速テキストチェックで CAPTCHA を検出。

| 検出対象 | 方法 | 対処 |
|---------|------|------|
| reCAPTCHA v2 | `.g-recaptcha` 要素 | ❌ スキップ |
| hCaptcha | `.h-captcha` 要素 | ❌ スキップ |
| Cloudflare Turnstile | `turnstile` テキスト | ❌ スキップ |
| Cloudflare 盾 | `verify you are human` | ❌ スキップ |

### 5. 営業文生成（DeepSeek）

企業情報をもとに DeepSeek API で営業文を生成。失敗時はテンプレートフォールバック。

### 6. Stagehand agent によるフォーム操作

Stagehand の `agent.execute()` がフォーム検出・入力・送信を自律実行。

```typescript
const agent = stagehand.agent({
  model: "deepseek/deepseek-chat",
  systemPrompt: "あなたはフォーム自動入力エージェントです...",
});
const result = await agent.execute({
  instruction: "お問い合わせフォームに以下の情報を入力して送信してください...",
  maxSteps: 15,
});
```

### 7. 結果判定 → Twenty CRM 書き戻し

| 結果 | CRM ステータス |
|------|---------------|
| 送信成功 | `送信済み / 未対応` |
| CAPTCHA | `ブロック / 未対応` |
| タイムアウト | `手動確認 / 未対応` |
| エラー | `手動確認 / 未対応` |

## 並列処理

Orchestrator が同時実行数を制御。

```
最大 8 並列
│
├─ Job 1: Dnb, Inc.          ████████░░░░  (処理中)
├─ Job 2: Example Corp        ████████████  (完了)
└─ Job 3...8: 空きスロット
```

- 上限 8 並列（DeepSeek API レート制限 + メモリ）
- 完了次第、次のジョブを即補充
- 全完了でコンテナが終了

---

## エラーハンドリングとリトライ

各ジョブは最大 3 回リトライ（指数バックオフ）。

```
1 回目: 通常実行
   ├─ 成功 → ステータス更新
   └─ 失敗 → 1 分待機
2 回目: リトライ
   ├─ 成功 → ステータス更新
   └─ 失敗 → 2 分待機
3 回目: 最終リトライ
   ├─ 成功 → ステータス更新
   └─ 失敗 →「手動確認」+ エラー記録
```

## Docker 実行モデル

**オンデマンドワンショット**。cron や常駐デーモンなし。

```
# Hermes から実行
docker run --rm --env-file .env outbound-form-agent:v2 --max 5

# 処理完了 → コンテナ即終了 → リソース解放
```

- スクリーンショット保存なし
- ログは 50MB×3 ファイル制限
- Chrome バイナリ以外の常駐データなし
